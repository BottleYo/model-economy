# 更新记录

本项目遵循语义化版本；显著变更记录在本文件。

## 0.6.0-rc.1 - 未发布候选版（2026-07-21 完成实现）

- 新增适合目录式安装的核心模式、可选六角色增强模式与失败关闭的降级状态，不改变本地配置/状态 schema。
- 新增只读 `status --format text|json` 接口，JSON schema 版本固定为 1，并明确身份均未验证。
- 准备技能型官方目录材料和 5 个正向、3 个负向用例，但未执行提交。

## 0.5.2 - 未发布里程碑（2026-07-21 完成实现）

- 新增双语静态 GitHub Pages、社区健康文件、支持/法律页面、Issue 模板、推广素材与 A/B 评估协议，不引入遥测或固定节省承诺。
- 扩展 manifest，补充公开发布者、仓库、网站、法律、Starter Prompt 与 PNG 截图元数据。
- 重写首次体验：默认推荐 inherited 增强、提供三类示例、明确停用/移除控制与跨设备指引。

## 0.5.1 - 2026-07-13

- 新增自包含的浅色、深色与输入框 SVG 品牌资产，便于在 Codex 插件设置页和输入框中识别 Model Economy。
- 使用纯色几何路由符号，不包含渐变、外部字体、脚本或内嵌位图。
- 增加 manifest 与资产契约测试，避免后续版本意外丢失图标或引入外部依赖。

## 0.5.0 - 2026-07-13

- 新增 `domain-context`、`module-design` 和 `disposable-prototype` 三个按需触发的轻量叶子 Skill。
- 叶子 Skill 不启动 subagent、不接管任务分类或模型路由，也不自行提交；正式实现继续由 Model Economy 单一编排。
- 为一次性原型增加隔离、禁止任何真实账户、外部写入、破坏性副作用和任何凭据来源的安全边界。
- 将轻量能力的触发条件与输出写入机器路由策略，并保持六角色和 `strong` 上限不变。
- 敏感内容扫描器允许精确的 GitHub 官方 `noreply@github.com` 合并提交身份，避免 PR 临时合并提交误报，同时继续拒绝其他域内地址和伪造后缀。

## 0.4.0 - 2026-07-11

- 用按风险触发的意图门、计划门、测试门和证据门替代对 Superpowers Skill 的运行依赖。
- 明确安装或启用 Superpowers 不构成完整流程授权；只有当前任务的显式 strict 请求才交出编排权。
- 简单任务不再强制设计文档或持久化计划，标准任务使用短计划，测试强度按行为风险调整。
- 将全局路由管理块升级为原生默认和单任务 strict 交权规则，同时保持对其他插件零修改。

## 0.3.0 - 2026-07-11

- 新增可选的 CodexBar 0.41.0 或更高版本适配器，用于查看本地 token 总量、模型拆分、项目筛选、日历窗口和估算成本。
- 为 JSON 和子进程增加资源边界，不直接读取 Codex 或 CodexBar 的凭据与 session 文件。
- 将标准任务的 explorer 与 reviewer 改为条件角色，增加每任务三次启动预算，并用机器策略表达主要执行者互斥。
- 明确 Model Economy 与 Superpowers 的单一编排优先级、不同审查证据层和最小委派上下文。
- `verify` 会拒绝旧插件版本生成的角色模板，直到完成升级。

## 0.2.0 - 2026-07-11

- 新增 `enable-global-routing` 与 `disable-global-routing`，以安全、幂等的方式管理全局 `AGENTS.md` 开发路由块。
- 扩展 `cost-aware-development` 的发现条件，使通用软件开发任务可以触发能力分层路由。
- 全局规则保持跨项目且不包含业务上下文；项目自身的 `AGENTS.md` 可以覆盖它。

## 0.1.0 - 2026-07-11

- 首次公开发布：提供能力分层角色路由、可替换模型档案和本地生命周期 CLI。
- 增加跨平台 CI、敏感内容检查和公开安全报告流程。
- 补充本机 0.1.0 预览评估，覆盖临时 HOME 生命周期、静态安装审阅、跨项目发现与公开报告边界。
- 为 `upgrade --dry-run` 提供生命周期集成验证入口。
- 扩大敏感内容扫描范围并对 Git 读取失败采取失败关闭，覆盖带注释标签。
- 事务回滚恢复原 POSIX 文件权限，并记录强制卸载的归属处理行为。
- 修复 Windows 默认代码页下的 UTF-8 处理问题。
