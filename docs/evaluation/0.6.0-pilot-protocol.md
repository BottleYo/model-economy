# Model Economy 0.6.0 推广期评估协议

首轮公开样本统一使用 `0.6.0`。若保留候选版历史样本，必须继续标记为 `0.6.0-rc.1`，不得与稳定版数据混合。

## 目标

在不进行角色级 Token 猜测的前提下，对比基线工作流与 Model Economy 的任务编排、用量和质量证据。首轮推广不等待本评估完成，也不发布固定节省比例。

## 样本

每个任务类别至少准备三个脱敏任务，基线组与 Model Economy 组使用相同目标、代码起点、验收标准和可用工具。机械任务必须满足全部五项确定性条件；大型/高风险任务记录是否具备增强角色隔离。

## 字段

使用同目录的 `0.6.0-pilot-template.csv`。Token 和成本来自可信平台界面或 CodexBar 白名单摘要；缺失值保持空白，不补零。`model_identity_verified` 与 `role_identity_verified` 在没有可信事件字段时必须为 `false`。

## 发布规则

- 至少完成每类三个可复算样本后，才讨论百分比。
- 报告样本数、环境、失败、缺失数据和测量限制。
- 不按时间窗口把 Token 归因到角色。
- 测试质量、审查发现或完成率下降时，不得以 Token 降低作为成功结论。
- 公开数据不得包含凭据、私人路径、原始 session、项目源码或内部 URL。
