# 0.1.0 Local A/B Evaluation

Date: 2026-07-11

## Summary

Model Economy 0.1.0 passed the local static readiness gates for a preview tag. The verified scope covers lifecycle installation, local marketplace registration, plugin installation, static discovery from two unrelated working directories, and local file integrity checks.

This report does not claim model routing effectiveness, role identity isolation, or token savings from live tasks. Those require real model-backed task samples and remain follow-up work.

## Static Gates

| Gate | Result | Evidence |
| --- | --- | --- |
| Temporary home lifecycle | Pass | `install --profile inherited`, `verify`, `upgrade --dry-run`, `uninstall --purge`, and purge check completed successfully. |
| Local marketplace registration | Pass | The local marketplace was registered and the plugin installed from `model-economy-public`. |
| Real home install | Pass | `install --profile openai-56` completed without conflicts. A repeated install was a no-op. |
| Static doctor | Pass | Static doctor completed successfully without `--smoke`. |

| Cross-project discovery | Pass | `plugin list --available --json` found the installed plugin from project A and temporary project B. |
| Cross-project static verify | Pass | Local `verify` passed from project A and temporary project B. |
| Test suite | Pass | Unit tests passed locally. |
| Sensitive-content scan | Pass | The repository scanner passed locally. |

## Cross-Project Scope

Project A was an unrelated existing working directory. Temporary project B was a minimal Git repository containing one public example Python file. No files were copied from project A into temporary project B.

The cross-project checks were intentionally static and model-free. They show that the installed plugin is visible through Codex plugin listing and that the local managed files remain valid from unrelated current working directories. They do not prove that a newly spawned model task will select a specific Model Economy role.

The success thresholds remain unchanged for the first real A/B run:

- Simple tasks should use zero strong calls.
- Standard tasks should use no more than one strong call.
- Large tasks should use no more than two strong calls.
- At least 70% of implementation steps should be completed by balanced/economy roles.
- Test quality should not be worse than the baseline.

Real token A/B values are intentionally not estimated here. They must come from reliable UI or platform usage data from real samples.

## Explicit Non-Claims

- No `--smoke` run was performed.
- No authentication files, session logs, or conversation transcripts were manually opened or parsed for this report.
- Static discovery does not prove model identity or role identity during a live model call.
- Token savings are not proven until real A/B samples are collected.
- Local task-8 orchestration artifacts remain ignored and are not part of this public report.

## Preview Decision

The static gates are sufficient for a local `v0.1.0` preview tag. Public repository publication should wait until the user reviews the preview and decides timing. Model-backed A/B evidence should be collected before claiming cost reduction.
