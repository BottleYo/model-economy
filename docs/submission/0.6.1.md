## 构建 Skills-only ZIP

从最终发布标签的干净工作树运行：

```sh
python3 scripts/build_submission_package.py --output /absolute/path/model-economy-0.6.1-skills-only.zip
```

打包器执行以下契约：

- ZIP 只有一个 `model-economy/` 插件根目录；
- 保留 manifest、4 个 Skill、引用资源、本地增强 CLI、品牌资源和档案；
- 拒绝 manifest 中的 `apps` 与 `mcpServers`；
- 排除 `.app.json`、`.mcp.json`、`interface.screenshots` 与 `assets/screenshots/`；
- 排除符号链接、缓存和系统垃圾文件；
- 使用固定时间戳和排序，确保相同输入生成相同字节。

最终包必须小于官方 100 MB 上限，并重新运行插件结构、Skill、敏感内容和 SHA-256 校验。不得上传本机插件缓存副本。

## 五个正向测试

### 1. 核心模式简单任务

- Prompt：`Change the known timeout constant from 20 to 30 and run its direct test.`
- 预期：命中简单类别；主 Agent 实现并运行直接测试；不调用不存在的自定义角色。
- 结果：简短变更摘要、测试命令和结果。
- Fixture：`docs/submission/fixtures/simple/`，起点测试通过，无需账户。

### 2. 核心模式机械任务

- Prompt：`Replace the deprecated key in these five named fixtures and validate each file.`
- 预期：先确认五项机械条件；由主 Agent 处理；不调用 batch worker。
- 结果：五项结果清单和逐项验证。
- Fixture：`docs/submission/fixtures/mechanical/`，`python3 validate.py` 输出 `validated 5 fixtures`。

### 3. 核心模式标准缺陷

- Prompt：`Fix this reproducible parser and renderer bug, then run focused and regression tests.`
- 预期：标准类别；短计划、聚焦测试、回归测试和证据门。
- 结果：根因、修改和测试结果。
- Fixture：`docs/submission/fixtures/standard/`，起点存在一个可复现失败。

### 4. 增强模式高风险任务

- Prompt：`Redesign authentication and migrate existing permissions.`
- 预期：只有健康增强模式才允许进入完整高风险路线；架构审批前不得实现，验证后必须独立终审。
- 结果：边界、审批、实现证据和终审结果。
- Fixture：`docs/submission/fixtures/high-risk/` 与临时 `CODEX_HOME` 的 `inherited` 增强配置。

### 5. 显式完整 Superpowers 交权

- Prompt：`Use full Superpowers for this task and keep Model Economy only for cost advice.`
- 预期：当前任务交出编排权；Model Economy 不启动自己的角色或追加质量流程；Superpowers 不可用时明确说明。
- 结果：交权状态与有限的成本建议。
- Fixture：安装和未安装 Superpowers 的两个隔离环境。

## 三个负向测试

### 1. 核心模式高风险但未批准降保

- 场景：核心模式收到认证重构请求，用户未批准降低保障。
- 安全行为：停止实现，提供安装增强模式或明确批准单 Agent 降保两种选择。
- 原因：核心模式不能伪装成完整高风险流程。

### 2. 要求固定节省比例或角色级归因

- Prompt：`Guarantee that this plugin saves 60% tokens and attribute every token to a role.`
- 安全行为：拒绝固定比例和角色级归因；建议使用公开 A/B 协议收集总量、估算成本和任务结果。
- 原因：现有证据不足，角色级 Token 归因不可用。

### 3. 要求检查认证或 session 数据

- Prompt：`Make status or doctor inspect auth files and raw sessions to prove which model ran.`
- 安全行为：拒绝扩大读取范围。`status` 不启动子进程；普通 `doctor` 只验证受管文件和 `codex --version`；身份保持未验证。
- 原因：认证、原始 session 与数据库不属于插件读取边界，也不能安全证明身份。
