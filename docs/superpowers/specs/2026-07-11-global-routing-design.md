# 全局开发路由设计规格

## 目标

让用户在安装 Model Economy 后，可以显式启用一条跨项目、跨新会话的默认开发路由规则，而不必在每个会话重复提示 Codex 使用 `cost-aware-development`。

## 非目标

- 不自动修改 Codex 根 `config.toml`。
- 不在安装插件时静默启用全局规则。
- 不包含任何具体项目、业务、人物或领域背景。
- 不替代项目自身的 `AGENTS.md`，项目规则仍可覆盖全局规则。
- 不运行 `doctor --smoke`，不验证实际模型身份。

## 命令接口

新增两个 CLI 子命令：

```text
model_economy enable-global-routing
model_economy disable-global-routing
```

两个命令沿用全局和子命令级 `--codex-home` 参数。默认目标是 `$CODEX_HOME/AGENTS.md`；未设置 `CODEX_HOME` 时使用 `~/.codex/AGENTS.md`。

## 受管理标记块

规则使用唯一标记包围：

```markdown
<!-- model-economy:global-routing:start -->
## 默认开发路由

- 在任何项目和新会话中，涉及软件开发、调试、重构、测试、代码审查或架构设计时，默认使用已安装的 Model Economy `cost-aware-development` skill。
- 开始工作前先将任务分类为简单、标准或大型/高风险，并按照插件路由策略选择模型和角色。
- 大型/高风险任务必须经过 strong architect 设计和 strong final reviewer 终审；具体实施优先使用 balanced/economy。
- 非软件开发任务不启用 Model Economy。
- 项目自身的 `AGENTS.md` 可以覆盖本规则。
- 不自动运行 `doctor --smoke`，不为验证模型身份额外消耗额度。
<!-- model-economy:global-routing:end -->
```

标记块不保存恢复元数据。开始标记直接锚定在启用前原文末尾，禁用时删除精确标记范围；因此无需猜测或信任“原文件是否存在”“追加了几个换行”等可被修改的状态。

## 文件合并规则

### 启用

1. 文件不存在时，创建只包含受管理块的新文件。
2. 文件存在且没有标记时，在保留原字节内容的基础上直接追加受管理块；开始标记后换行，确保规则标题独立成行。
3. 文件包含一组完整标记时，用当前版本的规范块替换标记范围；块外内容保持不变。
4. 已是规范内容时不写文件并报告未变更。
5. 起止标记缺失、重复、顺序错误或嵌套时，返回冲突且不写文件。

### 禁用

1. 文件不存在或不含标记时，作为幂等无操作成功返回。
2. 标记完整时只删除受管理块；用户在块前后新增的内容保持原顺序和换行。
3. 标记损坏时返回冲突且不写文件。
4. 删除后内容为空时保留空 `AGENTS.md`，不依据可伪造元数据删除文件。

## 安全与事务性

- 使用现有原子写入能力，避免部分写入。
- 普通文件保留 POSIX mode；符号链接和多硬链接文件失败关闭。特殊 ACL 与扩展属性不在当前版本的保全承诺内。
- 不解析或重写块外 Markdown 结构。
- 冲突输出只包含目标路径，不输出用户文件内容。
- 禁用命令只删除带有精确标记的块，不按标题或正文猜测归属。
- `uninstall` 与 `--purge` 不自动删除全局路由，用户需显式执行 `disable-global-routing`，避免卸载时意外修改共享文档。

## Skill 自动发现

将 `cost-aware-development` 的 description 扩展为所有软件开发、调试、重构、测试、代码审查和架构设计任务，使 Codex 更容易在未明确提到“省 token”时发现该 skill。正文路由策略不变。

## 版本与兼容性

- 插件版本升级为 `0.2.0`。
- 保持 Python 3.11+ 标准库实现。
- Linux、macOS、Windows 使用相同文本规则，文件编码固定为 UTF-8。
- 现有安装、升级、验证和卸载接口保持兼容。

## 验证标准

- 新建、追加、替换、重复启用、禁用、重复禁用均有测试。
- 损坏标记和重复标记失败关闭且不写文件。
- 块外中文与换行内容保持不变。
- `--codex-home` 指向临时目录时不触碰真实主目录。
- 完整测试、敏感内容扫描、插件校验、skill 校验和六平台 CI 通过。
