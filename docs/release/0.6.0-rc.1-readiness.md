# Model Economy 0.6.0-rc.1 发布就绪记录

检查时间：2026-07-21（Asia/Shanghai）

结论：仓库内候选版与本机 Codex CLI 已就绪，但远端 CI、Pages 和真实 marketplace 验收尚未完成，因此不得创建 tag 或 GitHub Pre-release。

## 已通过

- manifest、`pyproject.toml`、CLI 模板版本与契约测试统一为 `0.6.0-rc.1`。
- 插件 manifest 校验通过，版本同时符合 SemVer；`pyproject` 可按 PEP 440 规范化为 `0.6.0rc1`。
- 217 项单元/契约测试全部通过，其中包括当前 Codex CLI 的 smoke 命令兼容性检查，以及状态接口对叶子文件、父目录符号链接和多硬链接的失败关闭检查。
- 四个 Skill 校验、敏感内容扫描和 `git diff --check` 通过。
- 临时 `CODEX_HOME` 中核心模式、`inherited` 增强安装、本地验证、状态 JSON 与清除流程通过。
- `status` 在核心和增强模式下均报告 `plugin_version` 与模板版本为 `0.6.0-rc.1`，身份字段保持 `false`。
- 已从官方 npm registry 将本机 Codex CLI 修复并升级到 `0.144.6`，`codex --version` 与 `codex plugin --help` 通过。

## 尚未通过

- `https://bottleyo.github.io/model-economy/` 及 support/privacy/terms 三个子页面当前均返回 HTTP 404。
- GitHub Pages 站点尚未创建；Issues 已启用，Discussions 尚未启用。
- 真实 marketplace 安装、重复安装、升级与卸载必须在候选提交合并后的干净 checkout 中补跑。
- 当前工作树尚未提交、推送或创建 tag，CI 尚未在远端运行本次变更。

## 解除发布阻塞后的顺序

1. 审阅变更，创建 `0.6.0-rc.1` 独立提交并推送 PR。
2. 等待 PR CI 全绿并合并，同时启用 Discussions 与 GitHub Actions Pages。
3. 等待 main CI 与 Pages 部署成功，匿名检查四个公开 URL。
4. 从 merge SHA 的干净 checkout 补跑真实 marketplace 生命周期。
5. 创建并推送 `v0.6.0-rc.1` tag，等待 tag CI 全绿。
6. 创建 GitHub Pre-release，使用对应双语草案，再开始社交平台软启动。

不得在上述门槛完成前把当前候选版描述为正式 `0.6.0`，也不得宣传固定 Token 节省比例或已经验证模型/角色身份。
