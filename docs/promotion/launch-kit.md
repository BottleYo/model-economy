# Model Economy 推广素材包

## 统一定位

中文：让强模型负责关键判断，而不是日常机械工作。

English: Use strong models for decisions, not routine work.

统一限定：策略级上限控制请求范围；不承诺固定 Token 节省；不验证实际模型或角色身份；社区项目，不代表 OpenAI 官方产品。

## 演示分镜

1. 展示同一重流程处理所有任务造成的等待与编排开销。
2. 完成目录式安装，运行 `status` 展示无需六角色文件的核心模式。
3. 简单与标准任务由主 agent 完成，应用原生质量门。
4. 可选安装 `inherited` 档案，运行 `verify` 和 `status` 展示健康增强模式。
5. 高风险任务展示增强模式中的架构与终审关口；核心模式则明确失败关闭。
6. 以“无固定节省承诺、身份未验证、可停用与反馈入口”结束。

短版控制在 15 秒，完整演示控制在 60～90 秒；静音观看也必须能理解。

## 小红书六张 3:4 卡片

1. 痛点：一行修改也要长计划和多个 agent？
2. 方案：先判风险，再决定流程与能力档。
3. 路由：简单、机械、标准、大型/高风险。
4. 边界：策略级上限，不是账单保证。
5. 关系：默认原生质量门，显式请求才交给完整 Superpowers。
6. 行动：安装、查看示例、提交反馈。

发布前在小红书客户端预览裁切和字号，不嵌入私人项目、账号或额度截图。

## X 首帖草案

I built Model Economy, a community Codex plugin that classifies development work by risk before choosing workflow depth and model roles. Simple work stays simple; high-risk work keeps explicit architecture and review gates. Policy-level limits, no fixed token-savings claim.

## 中文首帖草案

我做了一个 Codex 社区插件 Model Economy：先判断开发任务风险，再决定需要多少流程、模型能力和 subagent。简单任务不堆仪式，高风险任务保留架构与终审关口。它使用策略级上限，不承诺固定 Token 节省，也不伪装已经验证模型身份。
