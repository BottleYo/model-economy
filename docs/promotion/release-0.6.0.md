# Model Economy 0.6.0 GitHub Release 文案

## English

Model Economy 0.6.0 is the first stable release of directory-safe cost-aware development routing for Codex.

Highlights:

- Core mode works immediately after a skills-only installation and does not depend on user-level custom roles.
- Optional enhanced mode adds six locally managed role definitions with inherited or explicit model mapping.
- Partial, conflicting, or outdated enhancement artifacts fail closed as degraded state.
- The read-only `status --format text|json` command reports mode and managed-file health without reading authentication or raw session data.
- Native intent, planning, testing, and evidence gates scale with task risk without requiring Superpowers.
- Bilingual Pages, support and legal pages, community templates, review fixtures, and a reproducible evaluation protocol are included.

The core and enhanced lifecycle has been validated through local marketplace installation, reinstall, upgrade, removal, and cross-platform CI. This release does not claim a fixed Token-saving percentage and does not verify model or role identity.

The skills-only Plugins Directory submission package is prepared but has not been submitted. Platform identity verification and formal review remain separate future steps.

## 简体中文

Model Economy 0.6.0 是首个稳定版，为 Codex 提供适合目录式安装的成本感知开发路由。

主要变化：

- 核心模式在仅安装 Skill 后即可运行，不依赖用户级自定义角色。
- 可选增强模式增加六个本地受管理角色，支持继承式或显式模型映射。
- 部分缺失、冲突或过期的增强配置会进入降级状态并失败关闭。
- 只读 `status --format text|json` 报告模式和受管理文件健康状态，不读取认证或原始 session 数据。
- 原生意图、计划、测试和证据质量门按任务风险伸缩，不依赖 Superpowers。
- 包含双语 Pages、支持与法律页面、社区模板、审核 fixture 和可复算评估协议。

核心与增强生命周期已经通过本地 marketplace 安装、重复安装、升级、移除和跨平台 CI 验证。本版本不承诺固定 Token 节省比例，也不验证模型或角色身份。

技能型官方 Plugins Directory 提交材料已经准备，但尚未提交；平台身份验证和正式审核仍是后续独立步骤。
